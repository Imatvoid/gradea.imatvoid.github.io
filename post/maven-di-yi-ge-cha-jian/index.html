Maven -- 第一个插件

2018-05-08

A Mojo is really just a goal in Maven, and plug-ins consist of any number of goals (Mojos).

如果对maven已经有了了解,并且英文不错的话请直接阅读maven文档

新建项目

在idea中新建一个插件项目

注意这里要保证maven的镜像配好,网络畅通,不然idea自动生成项目架构会有问题.

说明

可以在src包下找到MyMojo类,它继承了AbstractMojo,我们只需要实现execute方法就可以实现一个简单的插件.
idea默认已经为我们生成了一个demo

注意 @goal touch和@phase process-sources 实际是使用注释的方式指定的插件的一个goal(目标)和绑定的周期. 并不推荐这种方式 我们可以使用注解的方式.引入如下依赖:

 <dependency>
      <groupId>org.apache.maven.plugin-tools</groupId>
      <artifactId>maven-plugin-annotations</artifactId>
      <version>3.4</version>
    </dependency>

替换代码如下:

/**
 * Goal which touches a timestamp file.
 */
@Mojo(name = "touch", defaultPhase = LifecyclePhase.PROCESS_SOURCES)
public class MyMojo
        extends AbstractMojo {

    /**
     * Location of the output directory.
     **/
    @Parameter( property = "project.build.directory")
    private File outputDirectory;

    public void execute()
            throws MojoExecutionException {
        File f = outputDirectory;

在其他项目中测试

在自己的插件项目中install一下,安装到本地仓库
在另一个测试项目的POM中引入插件,配置goal和绑定周期.

 <build>
        <plugins>
            <plugin>
                <groupId>com.atvoid.maven.plugin</groupId>
                <artifactId>check-maven-conf</artifactId>
                <executions>
                    <execution>
                        <id>touch</id>
                        <phase>process-sources</phase>
                        <goals>
                            <goal>touch</goal>
                        </goals>
                    </execution>
                </executions>
            </plugin>
        </plugins>
    </build>

然后可以在idea右侧的maven列表选项里找到我们的插件

双击一下就会发现实现了在输出文件夹中生成一个touch文件的效果.

后续进阶我们可以实现一个校验配置文件的插件.
